# Stabi 产品规划（草案）

 > 版本：v0.1  
 > 目标读者：团队内部（产品、技术、运营）  
 > 更新时间：待补充

 ---

 ## 1. 产品定位与愿景

 ### 1.1 核心定位

 - **Stabi = 面向普通用户的收益体验层（Yield Experience Layer）**
 - 不是新的借贷协议 / AMM / 衍生品协议，而是：
   - 把 Aave / Pendle / Curve / LRT 等复杂协议产生的收益
   - 打包成 **简单、可理解、可选择的收益产品**

 ### 1.2 一句话介绍

 > Stabi 帮用户把“复杂的 DeFi 收益”变成“简单的理财产品”，提供稳定、可预期的链上收益体验。

 ### 1.3 我们解决的问题

 - **收益复杂**：底层协议多、概念多（APY 波动、PT/YT、杠杆、LRT…），普通用户难以理解。
 - **体验割裂**：用户需要在多个协议间跳转、手动操作、手动管理风险。
 - **缺乏稳定感**：DeFi 收益波动剧烈，用户安全感较低，不适合长期资金停留。
 - **缺少自有经济系统**：大多数“聚合器”只是把流量导向别的协议，自己没有独立的积分、权益与生态。

 ### 1.4 Stabi 的角色

 - **不是**：
   - Aave / Pendle / Curve 的替代品
   - 仅仅是一个策略聚合器 UI
 - **而是**：
   - 抽象底层协议 → 收益产品
   - 管理收益曲线 → 给用户“平滑”的体验
   - 设计自己的忠诚度与经济系统 → 积分、等级、未来权益

 ---

 ## 2. 目标用户与使用场景

 ### 2.1 目标用户画像

 - 对加密有基础认知，但不想研究复杂 DeFi 协议的用户：
   - CEX 理财 / Earn 用户
   - 稳定币持有者（USDC / USDT / DAI 等）
   - 想拿“被动收入”的 Web3 打工人
 - 小部分愿意参与更高收益产品的轻度风险用户：
   - 愿意尝试“增强收益”、“固定收益”等产品
   - 但不想理解 PT/YT、基差、期限结构等专业概念

 ### 2.2 核心使用场景

 - **场景 1：稳健理财**  
   用户只想把 USDC 放在一个“稳健、安全、年化 3–5%”的池子里，长期放置。
 - **场景 2：选择投资风格**  
   用户可以在“稳健 / 平衡 / 激进”几种收益套餐中选择，而不是面对几十个池子。
 - **场景 3：固定收益需求**  
   用户想锁定某个期限内的固定收益，比如“锁 3 个月，年化 ~X%”。
 - **场景 4：参与新玩法 / 活动池**  
   用户愿意用少量资金参与高收益 / 新协议合作池，获取更高收益和积分。

 ---

 ## 3. 产品结构与核心模块

 ### 3.1 产品总体结构

 从下到上分为六层：

 1. **底层收益引擎层（Base Yield Layer）**
 2. **收益路由层（Smart Yield Router）**
 3. **收益抽象层（Yield Abstraction Layer）**
 4. **收益体验层 / 产品层（Yield Experience & Product Layer）**
 5. **忠诚度与积分经济层（StabiPoints Layer）**
 6. **风险与收益曲线管理层（Risk & Smoothing Layer）**

 Stabi 的差异化价值主要集中在 **第 3～6 层**。

 ---

 ## 4. 核心产品设计

 ### 4.1 底层收益引擎（Base Yield Layer）

 **功能**：接入主流、安全、流动性好的收益来源协议，为上层产品提供基础收益。

 - 第一阶段必接：
   - **Aave**：稳定借贷收益来源 → 构建“稳健池”
 - 后续阶段接入：
   - **Pendle**：收益拆分 & 固定收益 / 增强收益 → 构建“固定收益池 / 增强池”
   - 其他可选协议：Curve、Compound、LRT/LST 协议等（待定）

 > 说明：仅接入协议并不会产生护城河，上层的抽象和产品设计才是关键。

 ---

 ### 4.2 收益路由层（Smart Yield Router）

 **目标**：在底层多个协议之间自动分配和调度资金，提升整体收益与风险控制能力。

 - 基本职责：
   - 收集各协议实时/预期 APY
   - 考虑风险评分与期限因素
   - 在 Aave / Pendle / 其他协议之间自动分配权重
   - 支持收益自动复投
 - 用户体验：
   - 用户只需选择“产品”（如稳健池），不需要关心背后路由逻辑。

 ---

 ### 4.3 收益抽象层（Yield Abstraction Layer）

 **目标**：把复杂的 DeFi 操作（尤其是 Pendle 的 PT/YT）封装成简单的操作按钮。

 - **抽象方向示例**：
   - 用户选择“固定收益产品” → 后台自动处理：
     - 选择合适期限的 PT
     - 处理到期前后的滚动、续期
   - 用户选择“增强收益产品” → 后台自动处理：
     - 分析 YT 收益 / 折价
     - 控制仓位与止损
 - 用户无需理解：
   - PT/YT 是什么
   - 折价、到期、滚动续期等细节

 ---

 ### 4.4 收益体验层 / 产品层（Yield Experience & Product Layer）

 **目标**：用**少量清晰的产品**替代“几十上百个池子”。

 初期计划提供 4–6 类核心产品：

 1. **稳健池（Stable Pool）**
    - 收益来源：以 Aave 等主流借贷为主
    - 目标年化：约 3–4%
    - 风险：低
    - 面向：主流用户 / 长期资金

 2. **平衡池（Balanced Pool）**
    - 收益来源：Aave + 适量 Pendle PT / 其他策略
    - 目标年化：约 4–6%
    - 风险：中等
    - 面向：愿意接受一定波动、追求更高收益的用户

 3. **增强池（Boost Pool）**
    - 收益来源：Pendle YT 等高收益策略
    - 目标年化：7–12%（浮动大）
    - 风险：偏高
    - 面向：小比例资金、轻度风险偏好用户

 4. **固定收益池（Fixed Yield Pool）**
    - 参考 Pendle PT，提供类似“锁定期限 + 预期固定收益”的产品体验：
      - 1 个月 / 3 个月 / 6 个月 / 12 个月
    - 对用户展示形式：
      - “锁 3 个月，目标年化 X%（范围）”
    - 对接 CEX 理财用户心智。

 5. **新手池（Beginner Pool）**
    - 主打：极易理解、极低风险、简单可用
    - 说明文案尽量简单（适合 DeFi 新手）

 6. **活动 / 季节性池（Campaign Pools）**
    - 用于合作、营销活动、新协议联动
    - 与 StabiPoints/空投/白名单等玩法绑定

 ## 5. 收益曲线稳定化（Smoothing APY）

 ### 5.1 问题

 - 底层协议 APY 大幅波动（周末高、工作日低，市场情绪波动等）。
 - 小白用户看到的是一条乱动的曲线，难以建立信任与长期预期。

 ### 5.2 设计思路

 - 为部分产品（如稳健池）设计「收益缓冲池」（Yield Buffer Pool）：
   - 实际收益 > 目标 APY 时：
     - 抽取部分超额收益注入缓冲池
   - 实际收益 < 目标 APY 时：
     - 从缓冲池拨出一部分补贴用户
 - 用户侧看到：
   - 一个较稳定、窄幅波动的 APY 区间（例如 4.0%–4.5%）

 ### 5.3 对用户与协议的价值

 - 用户：
   - 收益可预期，体验类似传统金融产品，更愿意长期持有。
 - 协议：
   - 通过波动收益的“剩余部分 + 缓冲池管理”获得利润空间。
   - 打造“收益银行”的标签，而不只是聚合器。

 ---

 ## 6. 忠诚度与经济系统（StabiPoints）

 ### 6.1 StabiPoints 概念

 - **StabiPoints = Stabi 内部积分 / 忠诚度单位**
 - 早期作为积分使用，后续可演进为权益凭证或与代币分配挂钩（遵守合规前提）。

 ### 6.2 获取规则（初步设想）

 - 存入资产即开始累积 StabiPoints：
   - 不同池子不同倍率（稳健池基础倍率，增强/固定池额外加成）
   - 随时间线性增长或按复利机制计分
 - 其他可计分行为：
   - 长期锁仓（锁定期越长，积分倍率越高）
   - 参与治理投票
   - 参与活动池
   - 推荐邀请（Referral）

 ### 6.3 积分用途（演进路线）

 - 阶段 1：
   - 积分只展示 & 记录，不立即赋予强权益（测试产品市场匹配）
 - 阶段 2：
   - 积分可用于手续费折扣 / APY 加成券 / 优先体验新产品等
 - 阶段 3：
   - 作为未来潜在代币空投 / 治理权重等分配依据之一

 ---

 ## 7. 风险与透明度

 ### 7.1 风险维度

 - 协议风险：底层 Aave / Pendle / 其他协议自身风险。
 - 智能合约风险：Stabi 自己的合约实现、路由逻辑、缓冲池逻辑。
 - 市场风险：收益波动、预期收益与实际收益偏离。
 - 流动性风险：用户取款高峰时的资金调度与缓冲能力。

 ### 7.2 风险管理原则

 - 优先接入成熟、审计充分的协议。
 - 逐步引入复杂功能（先从 Aave 稳健池开始）。
 - 收益稳定化逻辑从 Demo → 小规模实验 → 扩大范围。
 - 所有风险相关逻辑尽量链上透明，配合清晰的前端文案。

 ## 8. 产品迭代路线（高层级）

 ### Phase 0：基础 & 本地验证

 - 本地 + 测试网完成 Vault / Strategy 框架搭建与单元测试。
 - 完成文档（技术方案 + 产品规划）。

 ### Phase 1：Aave 稳健池 MVP

 - 集成 Aave（优先 Arbitrum 测试网 / 主网）。
 - 实现一个“稳健池”：
   - 存款 / 取款 / 利息累积 / 基础费率。
 - 提供最基础的 `StabiPoints` 记账逻辑（仅计分与展示）。

 ### Phase 2：收益稳定化 v0（Smoothing Demo）

 - 在稳健池上增加简单版收益缓冲机制。
 - 在测试网观察 APY 曲线、缓冲池行为。
 - 调整参数与风险边界。

 ### Phase 3：多池 & APY 套餐化

 - 新增平衡池 / 增强池 / 固定收益池等产品。
 - 引入 Pendle 作为收益引擎（从 PT → 固定收益开始）。
 - 前端/接口上以“产品”而非协议名称展示给用户。

 ### Phase 4：StabiPoints 进化 & 经济层建设

 - 扩展积分规则、用途与活动。
 - 设计 XP / 等级 / 徽章系统，增强用户粘性。
 - 规划潜在治理与代币相关路径（合规前提下）。

 ### Phase 5：高级功能与生态合作

 - 收益拆分小白版、内部资金调度系统（Yield OS）。
 - 引入更多协议与合作伙伴，构建 Stabi 生态。

 ## 10. Stabi 核心功能总览

下表从「功能模块 / 功能点 / 用户价值 / 特色与创新」四个维度，概括 Stabi 的核心能力，便于后续产品和开发对齐。

1. **稳定币存入 & 一键操作**  
   - **功能点**：支持 USDC / USDT / BUSD 等稳定币，一键存入、随时提现。  
   - **用户价值**：小白用户无需理解 DeFi 协议细节，即可获得链上收益。  
   - **特色 / 创新**：操作路径极简，UI 友好，降低使用门槛。

2. **策略池（Pool）选择**  
   - **功能点**：提供三类策略池：稳健 / 平衡 / 增强。  
   - **用户价值**：用户可按自身风险偏好选择合适产品，而不是在众多协议中迷失。  
   - **特色 / 创新**：小白可理解的 APY 套餐化设计，屏蔽底层复杂协议细节。

3. **收益缓冲池（Yield Buffer Pool）**  
   - **功能点**：高收益时将一部分收益注入 Buffer，低收益时从 Buffer 中补贴用户。  
   - **用户价值**：用户看到更稳定、平滑的 APY 曲线，体验接近传统金融理财。  
   - **特色 / 创新**：收益曲线平滑机制，为原创方向，传统 DeFi 协议尚未系统实现。

4. **自动收益路由 & 复投**  
   - **功能点**：后端智能调度 Aave、Pendle PT/YT 等协议资金，自动复投收益。  
   - **用户价值**：用户无需手动切换策略或复投，享受自动化收益优化。  
   - **特色 / 创新**：提升资金利用效率，降低操作复杂度，形成智能收益路由层。

5. **收益可视化与透明化**  
   - **功能点**：展示当前 APY、历史收益曲线、底层协议与资金分布。  
   - **用户价值**：用户可随时查看收益与资金流向，增强安全感。  
   - **特色 / 创新**：强调透明度与可审计性，为长期信任打基础。

6. **忠诚度积分系统（StabiPoints）**  
   - **功能点**：存款与使用行为累积积分，积分可兑换权益、提升 APY 或参与 DAO。  
   - **用户价值**：激励长期留存与社区参与，用户获得除收益外的额外权益。  
   - **特色 / 创新**：构建 Stabi 自有经济体系，不依赖单一底层协议的代币激励。

7. **风险与教育模块**  
   - **功能点**：展示各策略池的风险等级、底层协议风险、历史波动等信息。  
   - **用户价值**：帮助用户理解风险来源，做出更理性的配置决策，使用更安心。  
   - **特色 / 创新**：面向小白的教育向设计，降低认知门槛，把复杂风险语言翻译成易懂提示。
